# Общий подход
Для разработки использовал Claude Code (Opus 8). Рассказал ему про цель проекта и про бпоследующие шаги, вместе составили SPEC.md и CLAUDE.md. В качестве контекста Claude Code был доступен код предыдущего задания про локальные модели и код 03-weather-agent-tools с семинара.

# Веха 1. Цикл агента
Создан класс Agent, полем которого является история сообщений messages (массив словарей). В начале туда кладётся только системный промпт, в процессе работы ассистента туда дописываются сообщения пользователя и ответы ассистента. Весь массив messages каждый раз передаётся в LLM.

# Веха 2. Инструменты
Добавил агенту все перечисленные инструменты - Read, Write, Update, Delete, List, Execute. Их реализация и описание расположены в agent/tools.py. Есть массив `TOOLS`, элементами которого являются JSON-описания доступных инструментов. Каждый инструмент реализует самостоятельная функция, и поддерживается отображение (словарь) названий инструментов на функции-обработчики (`_HANDLERS`).

У меня не было проблем, когда агент ошибался с аргументами, но были другие проблемы:
1. У модели, которую я использовал в прошлом задании (qwen2.5-coder:7b) заявлена поддержка инструментов в API OpenAI, но не заработало. Claude порекомендовал перейти на другую модель, перешел на qwen2.5:7b-instruct. Я вспомнил, что на работе коллеги пытались настроить qwen-code для работы с моделью qwen-coder, развернутой на корпоративных серверах, и не получалось, очень может быть что по схожей причине
2. qwen2.5:7b-instruct оказалась флакающей в плане вызова инструментов и иногда вместо вызова инструмента через API OpenAI возвращает вызов инструмента текстом или вообще пустой ответ. Пришлось добавить детекцию пустого ответа и прасинг JSON для компенсации.

# Веха 3. AGENTS.md
Добавил поддержку. Для теста спросил агента "What's my name?" Без AGENTS.md ответ был
```
To determine your name, I would need some context or a file that contains it. You haven't provided any specific information about your name so far. Could you tell me more about what you're trying to achieve, or do you have a file where your name might be stored? If not, I'll assume we're just playing a game and call you "User" for now. What would you like to do next?
```
Затем создал AGENTS.md со следующим содержимым:
```
The user's name is Konstantin.
```
перезапустил и повторил вопрос. Теперь ответ был
```
Your name is Konstantin. How can I assist you today?
```

# Веха 4. Skills
Добавил поддержку скилов. В качестве примера, добавил скилл, в котором содержатся инструкции по оформлению названий коммитов. При запуске агент выводит список известных ему скиллов. Для теста написал промпт:
```
Please, commit staged changes
```
Агент успешно использовал скилл для формирования сообщения коммита.

# Веха 5. Субагент
Эксперимент проводил на такой задаче:
```shell
Use the task tool to delegate this to a subagent: create 6 files report1.txt … report6.txt, each containing a one-paragraph dummy description, then read all six back to verify them.
```
Использование субагента запросил явно. Также для проверки поддержал комманду `context`. Сразу после запуска агента у неё был такой вывод:
```
Context usage:
  system       1 msg   ≈    285 tok
  --------------------------------
  total        1 msg   ≈    285 tok   (0.9% of 32768)

```

После выполнения промпта:
```
Context usage:
  system       1 msg   ≈    285 tok
  user         1 msg   ≈     46 tok
  assistant    2 msg   ≈     84 tok
  tool         1 msg   ≈    109 tok
  --------------------------------
  total        5 msg   ≈    524 tok   (1.6% of 32768)
```
При перезапуске и явной просьбе не использовать инструмент `task` (т.е. не создавать субагента) вывод такой:
```
Context usage:
  system       1 msg   ≈    285 tok
  user         1 msg   ≈     40 tok
  assistant    3 msg   ≈    463 tok
  tool        18 msg   ≈    299 tok
  --------------------------------
  total       23 msg   ≈   1087 tok   (3.3% of 32768)
```
Видно, что в контексте появились результаты вызовов инструментов `read`/`write`.
